## （2）系统简介

### 系统名称
**WordMaster - 智能单词学习系统**

### 核心功能

WordMaster是一款专为英语学习者设计的智能化单词记忆应用，主要包含以下核心功能模块：

1. **单词学习模块**
   - 提供四级、六级、雅思、托福等多个权威词库
   - 每个单词包含音标、中文释义、英文例句及翻译
   - 支持单词发音播放（使用Web Speech API）
   - 三种学习反馈：不认识/模糊/认识，系统根据反馈调整复习计划

2. **单词测试模块**
   - 三种测试模式：中译英、英译中、混合模式
   - 智能选项生成，干扰项来自同词库的其他单词
   - 实时正确率统计和答题进度显示
   - 测试完成后提供详细的成绩分析和评价

3. **复习巩固模块**
   - 基于艾宾浩斯遗忘曲线原理设计复习算法
   - 智能计算每个单词的下次复习时间
   - 单词状态分类管理：新词、学习中、复习中、已掌握、困难词
   - 个性化复习计划，优先复习易遗忘单词

4. **学习进度模块**
   - 近7天学习趋势可视化图表
   - 单词掌握情况分类统计
   - 学习时长、测试次数、平均正确率等数据统计
   - 连续学习天数追踪（学习打卡功能）

5. **数据管理模块**
   - 使用LocalStorage实现本地数据持久化
   - 支持学习数据导入/导出（JSON格式）
   - 可自定义每日学习目标
   - 一键清除数据功能

### 目标用户

本系统主要面向以下用户群体：
- **大学生**: 备考英语四六级考试
- **留学预备人员**: 准备雅思、托福考试
- **英语爱好者**: 希望系统性提升词汇量
- **自学者**: 需要科学的学习方法和进度追踪

### 创新点

1. **智能化复习算法**: 基于艾宾浩斯遗忘曲线，结合用户对每个单词的掌握程度，动态调整复习间隔，实现个性化记忆规划。

2. **多维度学习反馈**: 不同于传统的"认识/不认识"二元反馈，系统提供"不认识/模糊/认识"三种状态，更精准地评估学习效果。

3. **数据可视化**: 通过图表直观展示学习趋势，帮助用户了解自己的学习模式和进步情况。

4. **纯前端架构**: 无需后端服务器和数据库，使用LocalStorage实现数据持久化，部署简单，随时随地可用。

5. **响应式设计**: 适配桌面和移动设备，支持触屏操作，满足不同场景下的学习需求。

## （3）应用技术

### 前端技术栈
| 技术 | 用途 |
|------|------|
| **HTML5** | 页面结构构建，语义化标签 |
| **CSS3** | 样式设计，动画效果，响应式布局 |
| **JavaScript (ES6+)** | 业务逻辑，交互功能，数据处理 |
| **Web Speech API** | 单词发音功能 |
| **LocalStorage API** | 本地数据持久化存储 |

### 核心技术特性

1. **CSS技术**
   - CSS Variables（自定义属性）实现主题管理
   - Flexbox和Grid布局实现响应式设计
   - CSS Transition和Animation实现平滑动画
   - 媒体查询（Media Queries）适配不同屏幕

2. **JavaScript技术**
   - ES6+语法：Class、Arrow Function、Destructuring等
   - 模块化设计，面向对象编程
   - 事件委托优化性能
   - 异步操作处理

3. **数据存储**
   - LocalStorage实现客户端数据持久化
   - JSON格式数据序列化/反序列化
   - 数据导入/导出功能

### 部署方式

- **本地运行**: 直接在浏览器中打开 `index.html` 文件
- **本地服务器**: 使用Python HTTP服务器 `python -m http.server 8080`
- **在线部署**: 可部署到GitHub Pages、Vercel、Netlify等静态托管平台

## （7）遇到的问题及解决方案

### 问题1：LocalStorage数据存储容量限制

**问题描述**: 
在开发过程中，我最初计划将所有词库的单词数据都存储在LocalStorage中。但在测试时发现，LocalStorage有5MB的存储限制，当单词数据量较大时会出现存储失败的情况。

**解决方案**:
1. **数据分离**: 将单词数据（wordDatabase）作为静态资源存储在data.js中，不存入LocalStorage
2. **只存状态**: LocalStorage仅存储用户的学习进度、单词状态、设置等动态数据
3. **数据压缩**: 对存储的数据进行精简，只保留必要字段

**学到的技巧**:
- LocalStorage适合存储小型结构化数据，不适合存储大量静态资源
- 合理设计数据模型，区分静态数据和动态数据
- 在使用本地存储前检查存储空间，做好异常处理

### 问题2：复习算法的智能性不足

**问题描述**:
最初的复习算法只是简单地按照时间顺序排列待复习单词，没有考虑用户对不同单词的掌握程度，导致复习效率不高。

**解决方案**:
1. **引入单词状态**: 为每个单词定义五种状态：new（新词）、learning（学习中）、reviewing（复习中）、mastered（已掌握）、difficult（困难词）
2. **差异化复习间隔**: 根据单词状态设置不同的复习间隔
   - learning状态：1天后复习
   - reviewing状态：3天后复习
   - difficult状态：1天后复习
3. **掌握度追踪**: 记录每个单词的复习次数和正确率，动态调整复习优先级

**学到的经验**:
- 算法设计需要考虑实际业务场景和用户需求
- 简单的规则往往比复杂的算法更有效
- 数据追踪是智能算法的基础，需要提前规划好数据模型

### 问题3：响应式布局的适配问题

**问题描述**:
在开发过程中，发现页面在不同尺寸的屏幕上显示效果差异较大，特别是在移动设备上，侧边栏和统计卡片的布局存在问题。

**解决方案**:
1. **移动优先设计**: 采用移动优先的响应式设计策略
2. **媒体查询**: 使用CSS媒体查询针对不同屏幕尺寸设置不同样式
3. **弹性布局**: 使用Flexbox和Grid布局替代固定像素布局
4. **侧边栏适配**: 在小屏幕上将侧边栏改为顶部导航栏

**学到的技巧**:
- 响应式设计需要在项目初期就规划好
- 使用相对单位（rem、%、vh/vw）而非固定像素
- 测试时要覆盖多种设备和屏幕尺寸
